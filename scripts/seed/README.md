# Seed Scripts

## Purpose

Contains synthetic data seeding for local and test environments.

## Owner

Developer experience and quality engineering.

## What Should Go Inside

- Synthetic users
- Synthetic wallets
- Test ledger accounts
- Provider mock data
- KYC test fixtures without real PII

## What Should Not Go Inside

- Real customer PII
- Production provider credentials
- Financial records copied from production
- Non-repeatable manual test data
