# Docs

## Purpose

Contains architecture, operational, security, compliance, and production readiness documentation.

## Owner

Architecture, platform engineering, SRE, security, and compliance.

## What Should Go Inside

- ADRs
- Diagrams
- API and event documentation
- Runbooks
- Compliance artifacts
- Incident and RCA documents

## What Should Not Go Inside

- Secrets
- Generated build artifacts
- Source-of-truth financial records
- Unapproved customer data exports
