# Production-readiness gaps

Each item has an owner recommendation and acceptance criteria. Severity is based on risk to customer funds, confidentiality, or recovery.

| ID / severity | Affected service | Production risk | Recommended fix | Acceptance criteria |
|---|---|---|---|---|
| PR-001 **critical** — Terraform parser errors | Terraform environments/modules | No reproducible infrastructure; CI plan/apply cannot be trusted. | Reformat and correct HCL blocks, run `fmt`, `init -backend=false`, `validate`, Checkov, tfsec and plan for all environments. | All three environments validate and plan from clean checkout with no parser errors; reviewed plan artifacts are retained. |
| PR-002 **critical** — Internal ledger API authorization not demonstrated | ledger-service, transaction-service, wallet-service | A reachable ledger endpoint could permit unauthorized account creation/posting/querying. | Put ledger behind private service discovery/mTLS or signed service credentials; add authorization and network policy tests; deny public ingress. | Unauthenticated and wrong-service calls return 401/403; only approved task role can post; penetration test evidence attached. |
| PR-003 **critical** — Financial E2E/concurrency/chaos evidence missing | transaction-service, ledger-service, wallet-service | Crash/retry races could still double debit or lose state despite unit tests. | Run database-backed E2E, concurrent transfer, restart-during-post, duplicate callback, provider timeout and reversal tests. | 100+ concurrent cases preserve ledger balance, one idempotent effect, valid terminal state, and no lost pending work. |
| PR-004 **critical** — Restore/DR not proven | RDS, S3, Secrets Manager, Terraform | A regional/data-loss event may exceed RTO/RPO or lose recovery secrets. | Execute encrypted RDS PITR/snapshot, S3 version, Terraform rebuild, secret recovery and DNS/traffic-shift drills. | Quarterly drill records measured RTO ≤1h/RPO ≤5m for core DB, reconciliation passes, and approved evidence exists. |
| PR-005 **high** — ECS edge wiring incomplete | ECS, ALB, API gateway, WAF | Tasks may be unreachable, unhealthy, or deployed without safe traffic draining. | Add/verify ALB listeners, target groups, health checks, service discovery, deployment circuit breakers, autoscaling and WAF association. | Staging has two healthy AZ-distributed tasks per critical service; unhealthy-task and rollback smoke tests pass. |
| PR-006 **high** — Terraform network security is too broad/unfinished | VPC, ECS, RDS, Redis | Broad egress and missing explicit ingress controls increase lateral movement/exfiltration risk. | Add SG-to-SG least-privilege rules, VPC endpoints/flow logs, service-specific egress and controlled admin access. | Checkov/tfsec pass or documented exceptions; flow logs show no public DB/Redis path. |
| PR-007 **high** — Production secrets injection/rotation is not wired end-to-end | ECS task definitions, Secrets Manager, KMS | Long-lived/default secrets or failed rotation can expose tokens and stop services. | Inject per-service secret ARNs, remove defaults in prod, add rotation/revocation and break-glass runbook tests. | No secret values in plan/log/image; rotation succeeds without downtime; compromised secret is revoked within SLA. |
| PR-008 **high** — Admin MFA/maker-checker runtime enforcement incomplete | admin-dashboard, auth-service, audit-service | Insider or compromised admin could freeze, reverse, or alter configuration without dual control. | Enforce MFA claims, distinct maker/checker identities, approval expiry, reason/before-after metadata and audit on searches/exports. | Automated authorization tests cover every sensitive action; self-approval fails; audit evidence is immutable and searchable. |
| PR-009 **high** — Scheduled reconciliation/settlement jobs not provisioned | reconciliation-service, SQS, EventBridge | Provider failures or settlement mismatches can remain hidden and balances unreconciled. | Provision hourly pending checks, daily settlement/ledger/suspense jobs, ownership/SLA, alerts and replay-safe workers. | Synthetic missing/amount/duplicate cases open, alert, assign, resolve and close with audited evidence. |
| PR-010 **high** — Provider trust boundary differs between paths | payment-integration-service, transaction-service | A forged/replayed callback path could transition a transaction incorrectly. | Route callbacks through one signature/timestamp/replay verifier; verify status/reference before settlement and add provider-specific contracts. | Invalid signature/timestamp/replay and unknown reference are rejected; duplicate is harmless; final settlement requires evidence. |
| PR-011 **high** — TLS/service identity runtime proof missing | all services, ALB, RDS, Redis | Credentials and PII may traverse plaintext internal links. | Enforce TLS to clients/providers/DB/Redis and mTLS or signed service auth internally; disable insecure endpoints. | TLS scans pass; plaintext connection test fails; certificates rotate before expiry. |
| PR-012 **medium** — Distributed rate limiting and fail-closed behavior unverified | API gateway, auth, Redis, WAF | Multi-task deployments can bypass local in-memory limits during abuse. | Use Redis atomic counters with bounded fallback, WAF/API quotas and explicit degraded-mode policy. | Login/OTP/PIN/transaction/callback abuse tests pass across two gateway instances. |
| PR-013 **medium** — Observability delivery and alert ownership unproven | all services, CloudWatch | Incidents may occur without actionable alerts or correlation across services. | Wire dashboards/alarms to SNS/on-call, validate metric dimensions, trace propagation and run alert fire drills. | Each critical alarm fires in staging, links to a runbook, pages an owner and has a recorded acknowledgement. |
| PR-014 **medium** — Load/performance capacity baseline absent | transaction, ledger, RDS, Redis, SQS | Peak traffic may exhaust connections, locks or queue age. | Define throughput/latency targets, run load tests, tune pools/autoscaling and cost model. | p95/p99 and error/queue/DB thresholds meet SLO at forecast peak plus headroom. |
| PR-015 **medium** — Backup retention/cross-region policy not environment-enforced | RDS, S3, Terraform | Accidental deletion or regional failure may exceed documented recovery objectives. | Pass explicit retention by env, snapshot copy/replication, object-lock decision and verify deletion protection. | Terraform plan shows dev/staging/prod policy differences and restore tests pass. |
| PR-016 **medium** — Cost budgets and cleanup automation lack real values/owners | CloudWatch, FinOps | Unexpected spend can grow unnoticed, especially NAT, logs, RDS and provider traffic. | Set per-env budgets, owner tags, anomaly alerts, schedules and monthly review. | Budget alarm test fires; monthly report identifies top drivers and cleanup actions. |
| PR-017 **low** — Architecture ADR/sequence evidence incomplete | docs/architecture | Operators/interviewers cannot verify design decisions or failure ordering. | Add ADRs and all required money/admin/provider sequence diagrams with versioned contracts. | Architecture index links every required diagram/ADR and owners approve it. |
| PR-018 **low** — Demo scripts assume endpoints/roles not always provisioned | scripts/demo, gateway, admin/audit | Demo may misrepresent production readiness or leak unsafe operational assumptions. | Pin mock setup, add endpoint capability checks and label expected failures. | Clean local run produces synthetic evidence without real credentials/data. |
