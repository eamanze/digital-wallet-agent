# Architecture Docs

## Purpose

Stores system architecture, service boundaries, diagrams, and architecture decisions.

## Owner

Architecture team.

## What Should Go Inside

- System context diagrams
- Service/container diagrams
- Sequence diagrams
- ADRs
- Data flow diagrams

## What Should Not Go Inside

- Implementation-only notes better kept in service folders
- Secrets
- Outdated diagrams without versioning
- Raw production data
