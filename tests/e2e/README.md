# E2E Tests

## Purpose

Validates complete user and operational journeys across services.

## Owner

Quality engineering.

## What Should Go Inside

- Register, KYC, fund, transfer, history flows
- Withdrawal and provider timeout flows
- Admin manual review flows
- Reconciliation exception flows

## What Should Not Go Inside

- Unit-level tests
- Production credentials
- Real customer PII
- Uncontrolled destructive tests
