# Security Tests

## Purpose

Contains automated security, abuse, and authorization tests.

## Owner

Security engineering and quality engineering.

## What Should Go Inside

- Auth bypass tests
- Rate limit tests
- OTP/PIN brute force tests
- Webhook replay tests
- Authorization tests
- Admin security tests

## What Should Not Go Inside

- Exploit payloads aimed at production without approval
- Secrets
- Real identity data
- Manual penetration test reports with sensitive details
