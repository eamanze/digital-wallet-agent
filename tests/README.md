# Tests

## Purpose

Contains cross-service and system-level tests for correctness, security, performance, and resilience.

## Owner

Quality engineering, service owners, and SRE.

## What Should Go Inside

- Integration tests
- Contract tests
- E2E tests
- Load tests
- Security tests
- Shared test fixtures

## What Should Not Go Inside

- Real customer PII
- Provider production credentials
- Tests that mutate production data
- Service source code
